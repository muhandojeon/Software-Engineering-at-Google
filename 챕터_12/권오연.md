* 단위 테스트는 단일 클래스나 메서드처럼 범위가 상대적으로 좁은 테스트
* 변하지 않는 테스트로 만들기 위해 노력하자
    * 순수 리팩터링 : 기존 테스트는 변경되지 않음
    * 새로운 기능 추가 : 기존 테스트는 변경되지 않음
    * 버그 수정 : 기존 테스트는 변경되지 않음
    * 행위 변경 : 기존 테스트가 변경!
* 상호작용이 아니라 상태를 테스트하자
    * 상태 테스트
    * 상호작용 테스트
        : 상태 테스트보다 깨지기 쉽다.
        : 결과가 무엇이냐가 중요한데, 상호작용 테스트는 결과에 도달하기 까지 시스템이 어떻게 작동하냐를 확인하려 듬
* 테스트는 DRY 가 아니라 DAMP
    * DAMP : 서술적이고 의미있는 문구
    * 다소 중복이 있더라도 서술적인 것이 테스트에서는 낫다.
